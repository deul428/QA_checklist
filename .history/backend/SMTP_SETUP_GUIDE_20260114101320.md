# 회사 메일 서버 설정 가이드

## 회사 도메인으로 메일 발송하기

### 방법 1: 회사 SMTP 서버 직접 사용 (권장)

`.env` 파일에 회사 SMTP 서버 정보를 설정하세요:

```env
# 회사 메일 서버 설정
SMTP_HOST=smtp.ajnet.co.kr  # 회사 SMTP 서버 주소 (IT 부서에 문의)
SMTP_PORT=587  # 또는 465 (SSL 사용 시)
SMTP_USER=your-email@ajnet.co.kr  # 회사 이메일 주소
SMTP_PASSWORD=회사_이메일_비밀번호  # 회사 이메일 비밀번호
SMTP_FROM_EMAIL=your-email@ajnet.co.kr  # 발신자 이메일 (보통 SMTP_USER와 동일)
SMTP_USE_SSL=false  # TLS 사용 시 false, SSL 사용 시 true
```

**일반적인 회사 메일 서버 설정:**
- **Microsoft Exchange/Office 365**: `smtp.office365.com`, 포트 587 (TLS)
- **Google Workspace**: `smtp.gmail.com`, 포트 587 (TLS)
- **일반 회사 서버**: `smtp.회사도메인`, 포트 587 또는 465

### 방법 2: Gmail에서 회사 도메인으로 발송 (제한적)

Gmail은 발신자 주소를 회사 도메인으로 변경할 수 없습니다. 
하지만 Gmail에서 회사 메일로 자동 포워딩 설정은 가능합니다:

1. Gmail 설정 → 전달 및 POP/IMAP
2. "이메일 전달" 설정
3. 회사 이메일 주소로 전달 설정

**단점**: 발신자가 여전히 Gmail 주소로 표시됨

### 방법 3: 회사 메일 서버 정보 확인 방법

1. **IT 부서에 문의**
   - SMTP 서버 주소
   - 포트 번호 (587 또는 465)
   - 인증 방식
   - 방화벽/보안 정책

2. **Outlook/Thunderbird 설정 확인**
   - 기존 메일 클라이언트 설정에서 SMTP 정보 확인
   - 서버 주소와 포트 번호 확인

3. **일반적인 회사 메일 서버 주소**
   - `smtp.ajnet.co.kr`
   - `mail.ajnet.co.kr`
   - `smtp-mail.ajnet.co.kr`

## SSL/TLS 설정

### TLS 사용 (포트 587) - 권장
```env
SMTP_HOST=smtp.ajnet.co.kr
SMTP_PORT=587
SMTP_USE_SSL=false
```

### SSL 사용 (포트 465)
```env
SMTP_HOST=smtp.ajnet.co.kr
SMTP_PORT=465
SMTP_USE_SSL=true
```

## 테스트 방법

설정 후 다음 명령어로 테스트하세요:

```bash
python test_email.py
```

## 문제 해결

### 1. 연결 실패
- 방화벽 설정 확인
- 회사 네트워크에서만 접근 가능한지 확인
- VPN 연결 필요 여부 확인

### 2. 인증 실패
- 이메일 주소와 비밀번호 확인
- 앱 비밀번호 사용 필요 여부 확인
- 2단계 인증 설정 확인

### 3. 메일이 스팸으로 분류됨
- SPF, DKIM, DMARC 레코드 설정 확인
- IT 부서에 문의하여 DNS 레코드 확인
